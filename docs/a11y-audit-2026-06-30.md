4. **Upgrade Nav/Tab from Regular to SemiBold**
5. **Add `allowFontScaling={true}` audit** — confirm no component disables it
6. **Replace JetBrains Mono** with IBM Plex Sans (SemiBold for timers) across all non-code screens
7. **Add `accessibilityRole="header"`** to all Label/Overline category separators

### Critical Issues

**CRIT-01 — FilterChip/selected width: 43px**  
1px under the HIG 44pt and WCAG 2.5.5 44px floor. FilterChips are interactive toggles — touch targets apply. The selected variant is narrower because its label text is shorter and padding isn't compensating.  
**Fix:** Enforce `minWidth: 44` and `minHeight: 44` regardless of label length.

**CRIT-02 — TopAppBar icon buttons: estimated 24×24px**  
Container (390×102) is irrelevant — individual icon Pressable wrappers are almost certainly 24px square (the icon size), which is a hard HIG and WCAG 2.5.5 failure.  
**Fix:**
```jsx
<Pressable style={{ width: 44, height: 44, alignItems: 'center', justifyContent: 'center' }}>
  <BackArrowIcon size={24} />
</Pressable>
```

### Warnings

**WARN-01 — SearchBar at exactly 44px:**  
Zero margin — keyboard layout shift or border-stroke inclusion can push the effective area below threshold.  
**Fix:** Set `minHeight: 48`.

**WARN-02 — Button height drift 50→56px:**  
Acceptable now; creates future regression vector. Consolidate to `buttonHeight = 56` token.

**WARN-03 — FilterChip/StatusTag actual rendered heights unknown:**  
The 100px Figma values are specimen containers. Measure on device before declaring compliance.

1. **Focus ring system** — 2px solid `#009540`, 2px offset on all interactive components + `:focus-visible` CSS in WebView screens

| #c2d832 lime not bound to token in v1 | DS gap — binding incomplete |
| #fcab29 hardcoded in 7 screens | Implementation deviation — value matches, binding doesn't |
| #f9a826 warm accent | DS gap — no token defined; pure rogue |

| No focus rings designed | DS gap — affects every component |
| Label/Overline at 10px | DS gap |
| All AUTO line heights | DS gap |

> **Most actionable insight:** Two of the three highest-severity failures (secondary text and primary green for text) are **design system gaps**, not implementation errors. Fixing only deviating screens without correcting the canonical token values will leave the app in a 1.4.3 failure state even if every component perfectly implements the tokens.
