# Admin dashboard UX audit — 2026-07-22

**Persona:** Donna — sole program admin; laptop + iPad; job is clear session reviews, protect court-hour deadlines, then occasional ops (orders, events, feedback).  
**Method:** Review against PRD §7.2 + goals G1–G5; scan of operational dashboard patterns (action-first admin UX, Stripe/Linear progressive disclosure, work-queue tables).  
**Surface:** `admin/` home (`DashboardWorkbench`).

## Research takeaways applied

| Principle | Source pattern | Donna implication |
|-----------|----------------|-------------------|
| Next action in ~10s | Role-based ops dashboards | Lead with “what’s waiting” + one **Review next** CTA |
| Action > decoration | Admin UX 2026 / table-first tools | Charts and maps are optional; queue is the work |
| KPI → evidence | Stripe / ops dashboards | Keep KPI cards linked, but **after** the queue |
| Progressive disclosure | Linear / Stripe | Collapse neighborhood map + charts behind Show |
| Calm hierarchy | Stripe type scale | “Today” briefing strip, then amber queue only |
| Bulk on tables | Moderation / CBO work queues | Select + **Approve selected** |
| Mobile decisions | iPad-first PRD | Sticky **Review next** above bottom nav |

## Findings → changes

| # | Finding (Donna’s pain) | Change shipped |
|---|------------------------|----------------|
| 1 | “Dashboard” + KPI row stole first attention from the queue | Rename page to **Today**; **TodayFocus** strip with count + primary CTA |
| 2 | Queue lacked urgency / duration / court clarity | Waiting-age labels, duration, **Court** chip, **Review** action |
| 3 | No bulk path for obvious approvals | Select all / row checkboxes + **Approve selected** |
| 4 | Heatmap competed with daily ops | Collapsed under **Neighborhood map · optional** |
| 5 | Recent feed duplicated queue rows | Default recent feed **skips** items still in queue |
| 6 | Court risk cards looked neutral | Behind vs due-soon color; days left / overdue |
| 7 | Keyboard power-user tip was noisy on the page | Tip moved under queue; full shortcuts stay in drawer |
| 8 | Mobile lost the primary CTA while scrolling | Sticky **Review next** bar above bottom nav |

## Intentional non-changes

- Mock banner stays (honesty > polish while fixtures are on).
- Period toggle kept (hours/KPIs still period-scoped).
- Donuts remain available under Charts — Donna needs them for board updates, not every morning.

## Follow-up (2026-07-22) — Bento simplification

Dashboard rebuilt as a **bento grid** to cut clutter:

| Tile | Role |
|------|------|
| Review (2×2) | Hero work surface — Start + Approve/Review rows |
| Waiting / Approved / Hours / Feedback | One-metric tiles → deep links |
| Court hours | Deadline list |
| Jump to | Common destinations |
| Insights (collapsed) | All charts + map in one place |

Removed from the default surface: TodayFocus strip, bulk select, recent activity feed, sticky mobile CTA, separate map/charts disclosures.
