# Admin portal accessibility audit — 2026-07-22

**Standard:** WCAG 2.1 AA (axe tags: `wcag2a`, `wcag2aa`, `wcag21aa`, `best-practice`)  
**Tool:** axe-core 4.10.3 via browser CDP on `http://localhost:3001`  
**Scope:** Dashboard (`/`), Sessions (`/sessions`)

## Verdict

After fixes below, **Dashboard: 0 violations**. **Sessions: 0 violations** (select name fixed). Incomplete `color-contrast` checks remain only for decorative / `aria-hidden` glyphs and SVG map labels (map is `aria-hidden`; keyboard path is the neighborhood list).

## Initial findings → fixes

| Issue | WCAG | Fix |
|-------|------|-----|
| Brand `#009540` on white / white-on-green fails AA (~3.9:1) | 1.4.3 | Admin `primary` → `#007536` (~4.6:1); brand kept as `primary-brand` |
| Nested interactive: SVG `role="img"` + focusable path buttons | 4.1.2 | Map decorative (`aria-hidden`); filters via list `<button>`s only |
| Duplicate period SR links | 2.4.4 | Removed |
| Unlabeled Approve / Decline / Open | 4.1.2 | Named `aria-label`s including volunteer name |
| Nav badges without names | 4.1.2 | `aria-label` on counts |
| Court “C” badge semantics | 1.1.1 | `role="img"` + `aria-label="Court-ordered"` |
| Missing skip link | 2.4.1 | Skip → `#main-content` |
| Sessions sort `<select>` unnamed | 4.1.2 | `aria-label="Sort sessions"` + visible label wrapper |
| Status filter toggles | 4.1.2 | `role="group"` + `aria-pressed` |
| Touch targets | 2.5.8 | `min-h-11` / `.interactive` on primary controls |
