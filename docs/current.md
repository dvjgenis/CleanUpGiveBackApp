## Sessions + geolocation (implemented ‚Äî Expo Go test phase)

- **Supabase anonymous auth** on app launch via `AuthProvider` in `_layout.tsx` (`frontend/src/lib/supabase.ts`)
- **Fly sessions API** client (`frontend/src/lib/api.ts`, `sessionsApi.ts`) ‚Äî persists create/checkpoint/finalize when `EXPO_PUBLIC_API_URL` is set
- **GPS + camera** live in Expo Go via `liveSessionStore` (`expo-location` BestForNavigation at 1s interval; 2D Kalman position filter; foreground/background dedupe; hardened capture filters ‚Äî stationary detection, accuracy-adaptive min-move, gap recovery, sharp-reversal rejection; device-compass heading with adaptive EMA + platform accuracy gating and GPS fallback; EMA-smoothed live arrow; optional Follow toggle; lighter live display simplify + tip segment on maps; Douglas-Peucker on previews/replay; live maps wait for first GPS fix with ‚ÄúGetting precise location‚Ä¶‚Äù instead of US overview flash). **Live session draft** persists to AsyncStorage while active; **Resume/Discard** on cold start via `LiveSessionResumeGate`. **Background GPS** during active sessions via `expo-task-manager` when Always location is granted (EAS dev build required; **Expo Go is foreground-only** ‚Äî live tracker shows a soft banner and restarts GPS when the app returns to foreground; route gaps while locked/backgrounded are expected)
- **WebView map** in Expo Go (MapLibre GL JS + Carto Voyager / Dark Matter) per ADR-005; user pan/zoom on live and preview maps; live tracker map layer picker (Standard / Satellite / Hybrid) wired to `liveSessionStore.mapLayer`; Standard light/dark theme via `mapThemeStore` (auto by local time 7pm‚Äì6am, Account toggle to follow time of day, live sun/moon tool to override); **tracker UI chrome** (timer/location/tools/checkpoint/CTAs/Map Types) follows the same theme via `trackerChromeTheme.ts`; dark theme repaints parks/nature reserves/green landcover a legible dark green (upstream Dark Matter renders them invisible against the near-black background ‚Äî see `mapStyles.ts`); weather pill uses Open-Meteo `weather_code` icons; native MapLibre unchanged for EAS builds
- **WebView map** in Expo Go (MapLibre GL JS + Carto Voyager / Dark Matter) per ADR-005; user pan/zoom on live and preview maps; live tracker map layer picker (Standard / Satellite / Hybrid) wired to `liveSessionStore.mapLayer`; Standard light/dark theme via `mapThemeStore` (auto by local time 7pm‚Äì6am, Account toggle to follow time of day, live sun/moon tool to override); dark theme repaints parks/nature reserves/green landcover a legible dark green (upstream Dark Matter renders them invisible against the near-black background ‚Äî see `mapStyles.ts`); weather pill uses Open-Meteo `weather_code` icons; native MapLibre unchanged for EAS builds

**API:** `EXPO_PUBLIC_API_URL=https://cleanup-sessions.fly.dev` in `frontend/.env`. Backend needs `DATABASE_URL` + `SUPABASE_URL` on Fly.

## How to run

```bash
npm install --prefix frontend
npm start              # default: tunnel (Wi‚ÄëFi / hotspot / cellular)
npm run start:lan      # same Wi‚ÄëFi only (fast)
npm run start:device   # tunnel (alias)
```

Full matrix and troubleshooting: [expo-go-dev-networking.md](frontend/specs/expo-go-dev-networking.md).

**Physical device (Expo Go):** **`npm run start:device`** when the phone is on **LTE/5G** and the Mac is on Wi‚ÄëFi. **`npm start`** auto-tunnels on **iPhone Personal Hotspot** (`172.20.10.x`). On shared home Wi‚ÄëFi use **`npm run start:lan`**. Metro clears inherited `CI=true` so fast refresh stays enabled.

### EAS development build (background GPS + native MapLibre)

See [accounts-and-access.md](accounts-and-access.md). After installing a development build: `cd frontend && npm run start:dev-client`. **Rebuild required** after `app.json` / `expo-task-manager` plugin changes. Checkpoint photos use **`expo-camera`** in both Expo Go and the dev client.
