## Sessions + geolocation (implemented тАФ Expo Go test phase)

- **Supabase anonymous auth** on app launch via `AuthProvider` in `_layout.tsx` (`frontend/src/lib/supabase.ts`)
- **Fly sessions API** client (`frontend/src/lib/api.ts`, `sessionsApi.ts`) тАФ persists create/checkpoint/finalize when `EXPO_PUBLIC_API_URL` is set
- **GPS + camera** live in Expo Go via `liveSessionStore` (`expo-location` BestForNavigation at 1s interval; 2D Kalman position filter; hardened capture filters тАФ stationary detection, accuracy-adaptive min-move, gap recovery, sharp-reversal rejection; device-compass heading with GPS fallback; EMA-smoothed live arrow; optional Follow toggle; Douglas-Peucker display simplification on maps). **Background GPS** during active sessions via `expo-task-manager` when Always location is granted (EAS dev build required; foreground-only in Expo Go)
- **WebView map** in Expo Go (MapLibre GL JS + Carto Voyager / Dark Matter) per ADR-005; user pan/zoom on live and preview maps; live tracker map layer picker (Standard / Satellite / Hybrid) wired to `liveSessionStore.mapLayer`; Standard light/dark theme via `mapThemeStore` (auto by local time 7pmтАУ6am, Account toggle to follow time of day, live sun/moon tool to override); dark theme repaints parks/nature reserves/green landcover a legible dark green (upstream Dark Matter renders them invisible against the near-black background тАФ see `mapStyles.ts`); weather pill uses Open-Meteo `weather_code` icons; native MapLibre unchanged for EAS builds
- **Sessions list** fetches from API when configured; falls back to mocks otherwise
- **Backend:** Fastify + Prisma in `backend/sessions/` тАФ schema pushed to Supabase; Fly deploy pending org machine limit
- **Sessions list** fetches from Fly API when `EXPO_PUBLIC_API_URL` is set тАФ loading/empty/error states; no placeholder rows
- **Session detail** fetches from Fly API + Supabase Storage signed URLs via `useSessionDetail`
- **Backend:** Fastify + Prisma in `backend/sessions/` тАФ deployed to `https://cleanup-sessions.fly.dev`; uses Supabase Postgres + JWKS (ES256) auth verification

**API:** `EXPO_PUBLIC_API_URL=https://cleanup-sessions.fly.dev` in `frontend/.env`. Backend needs `DATABASE_URL` + `SUPABASE_URL` on Fly.

See [accounts-and-access.md](accounts-and-access.md). After installing a development build: `cd frontend && npm run start:dev-client`. **Rebuild required** after `app.json` / `expo-task-manager` / VisionCamera plugin changes.
